macOS 下 Sublime Emmet 嵌套表达Tab键不起作用的问题

发表于 2016-12-04   |   分类于 Tool

Emmet

广大Web前端开发者们对 Emmet 插件一定不会陌生，它使用仿CSS选择器的语法来生成代码，大大提高了HTML/CSS代码编写的速度。

缩写：ul>li.item$*3

安装方法也很简单

  1. ⇪ ⌘ P打开并输入“Package Control: Install Package”，找到并安装Emmet。
  2. 重启Sublime。

问题

不过我在MAC下安装后出现嵌套表达然后按Tab键不能实现作用的问题，对此记录解决方法如下文所述。

首先检查，确保已经命名了文件名后缀为html，如果确定编译环境为html，那么可能是PyV8的问题。我在安装Emmet后貌似自动弹出了提示警告PyV8有问题，但没在意，后来果真问题就出现在这。对此官网上也有说明：

Warning: this plugin may not work at all in some OSes since it written in JavaScript and uses PyV8 and Google V8binaries to run.

后来再回到Github查找原因时才发现这么一段话：

WARNING: When plugin is installed, it will automatically download required PyV8 binary so you have to wait a bit (see Loading PyV8 binary message on status bar).

手动安装PyV8

至于PyV8库，其实就是利用v8引擎让python来直接与javascript互操作。既然自动安装出了问题，那就只能手动安装了。

  1. 下载For Sublime Text 3 (Python 3.3)对应MACos的版本： OSX 10.7+；
  2. 打开Sublime，选择Preferences ->Browse Packages...；
  3. 然后在Finder下打开此时与“Packages”同级目录的“Installed Packages”，将解压后的PyV8文件夹拷贝到此路径下即可。
  4. 重启Sublime。

测试

ul>li.item$*5

至此，问题解决。
